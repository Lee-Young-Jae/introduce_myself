Part 2

React의 장점
  • ✔ 컴포넌트 단위의 코드 작성
  • ✔ 가상 DOM
  • ✔ 넓은 커뮤니티
  • ✔ React Native

React 는 페이스북에서 개발한 오픈소스 프레임워크로

JavaScript를 기반으로 사용자 인터페이스를 구현하는

현 시점에서 가장 인기가 많은 웹 프론트엔드 개발 도구입니다.

동적인 웹페이지가 많아지는 요즘 많은 유저 인터렉션을 처리하기 좋은 프론트엔드 프레임워크로서 가장 인기가 많은 React.js 를 주제로 선정하게 되었습니다.

2021 StackOverFlow 개발자 설문조사의 웹 프레임워크 부분에서
React의 사용량이 1위에 올라 있는 것을 볼 수 있습니다.

국내의 여러 기업들 역시 실험적인 기능들과 이벤트 페이지 등을 작성할 때 개발이 빠른 Web기술을 주로 사용하기 때문에 팀 내부에 react등을 사용하는 프론트엔드 팀을 따로 운용한다고 합니다.

React 는 컴포넌트 단위로 코드를 작성하기 때문에 높은 재사용성과 용이한 유지보수를 제공해 줍니다.

또한 가상 DOM 개념을 통해서 가령 페이지 내용에 변화가 생겨 DOM 전체가 다시 렌더링되는 전통적인 웹페이지와 다르게 State가 변화한 부분만 렌더링 하기 때문에
브라우저의 연산 비용을 낮추고 퍼포먼스를 높여줍니다. 결국 유저와의 인터렉션 부분에서 이점을 가져옵니다.

프레임워크 자체의 인기가 높은 만큼 사용자들과 커뮤니티를 통해 얻을 수 있는 정보가 많고 Facebook 이라는 대기업의 꾸준한 지원이 장점입니다.

마지막으로, React만 공부해도 문법이 비슷한 React Native를 상대적으로 쉽게 배울 수 있습니다. React Native는 ios, android 등 플랫폼에 구애받지 않고 하이브리드 앱 어플리케이션을 개발하는 프레임워크 입니다.

React 는 공식 웹페이지 https://ko.reactjs.org/의
튜토리얼을 통해 쉽고 빠르게 배울 수 있습니다.
